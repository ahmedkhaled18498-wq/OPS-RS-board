# Ops Task Board — توثيق كامل للتول واللوجيك (Updated)

ملف واحد بيشرح كل حاجة في `ops-task-board-v2.html`: الأدوار، البيانات، التخزين، وكل دالة (function) بتعمل إيه بالظبط.

> **آخر تحديثين:**
> 1. هيدر ملف الـ Export بقى بالإنجليزي بالكامل.
> 2. لما تكتب ID بيبدأ بحرف **W** في خانة تسجيل الدخول، بيظهر سكرول بار فيه أرقام الكرينز (C01–C07) — تختار منه قبل ما تدخل، والاختيار ده بيتكتب تلقائي في خانة Crane لأي تاسك جديد تفتحه بعد كده.

---

## 1. الفكرة العامة

لوحة تاسكات (Task Board) شغالة بالكامل من غير سيرفر — كل حاجة بتتخزن في **localStorage** بتاع المتصفح. بيدخلها 4 أنواع مستخدمين بأدوار مختلفة، وكل دور بيشوف واجهة مختلفة حسب صلاحياته.

مفيش باك إند ومفيش قاعدة بيانات حقيقية؛ يعني البيانات محفوظة على **الجهاز/المتصفح** اللي اتفتحت منه بس. لو فتحت الصفحة من جهاز تاني أو متصفح تاني (كروم بدل سفاري مثلاً) هتلاقي بيانات مختلفة، لأن كل نسخة ليها localStorage خاص بيها.

---

## 2. الأدوار (Roles)

الـ ID بتاع أي مستخدم لازم يكون بالصيغة: **حرف واحد (W/P/M/V) + 4 أرقام**، مثال: `W1201`.

| الحرف | الاسم | الصلاحيات |
|---|---|---|
| **W** | D&W (Data & Warehouse) | بينشئ تاسكات جديدة (Container + Position + Crane + Truck)، ويقفل (Done) التاسك بتاعه لما يبقى In Progress، ويقدر يحذف تاسكاته هو بس. |
| **P** | Port / Planner | بيشوف كل التاسكات اللي حالتها Waiting، وبيدوس Start بعد ما يكتب Position on Arrival، وبيشوف التاسكات اللي هو بدأها (In Progress). |
| **M** | Monitor | صلاحيات كاملة: بيشوف **كل** التاسكات + **History دائم لكل الحاويات حتى اللي اتحذفت**، وبيقدر يحذف أي تاسك، وعنده زرار Export لتحميل شيت Excel/CSV. |
| **V** | Viewer | مشاهدة فقط للوحة الحية (التاسكات الموجودة دلوقتي)، من غير أي تعديل أو حذف. |

الدالة `validId(v)` هي اللي بتتأكد إن الـ ID متطابق مع الـ regex: `/^[WPMV]\d{4}$/`.

## 3. تسجيل الدخول (Login)

- الـ ID بيتخزن في `localStorage` تحت مفتاح `ops-task-board-v2:my-id`.
- `loadId()`: بتقرأ الـ ID المحفوظ لو موجود وصالح، وتحدد `myRole` من أول حرف فيه. كمان بتقرأ آخر كرين محفوظ (`preselectedCrane`) من مفتاح `ops-task-board-v2:pending-crane`.
- `saveId(v)`: بتتحقق من صحة الـ ID، تحفظه، وتعمل `loadTasks()` + `render()` من جديد.
- زرار "خروج" بيمسح `myId`/`myRole` من الذاكرة (من غير ما يمسح الـ localStorage)، فلو عمل ريفريش هيرجع يدخل بنفس الـ ID تاني.

### 3.1 كرين بيكر قبل الدخول (جديد)

بينما المستخدم لسه بيكتب في خانة الـ ID (قبل ما يدوس "دخول")، فيه `input` event listener بيراقب كل حرف بيتكتب عن طريق `toggleCranePicker(val)`:

- لو أول حرف في الكتابة (بعد ما تتحول لحروف كبيرة) هو **W**، بيظهر `div` باسم `#cranePickerWrap` تحت خانة الإدخال مباشرة (بتصميم dropdown عايم فوق الصفحة، مش بياخد مساحة من الـ layout).
- جوه الـ div ده بيتنادى `renderCranePicker()`، اللي بترسم سكرول بار أفقي (`.crane-picker`) فيه 7 أزرار: **C01, C02, ... C07**.
- لو الحرف الأول مش W (يعني P أو M أو V أو أي حاجة تانية)، الـ picker بيختفي تاني (`display:none`).

لما المستخدم يدوس على أي كرين من السكرول بار:
1. القيمة بتتحفظ في متغير `preselectedCrane`.
2. بتتحفظ كمان في `localStorage` تحت مفتاح `ops-task-board-v2:pending-crane` — يعني تفضل محفوظة حتى لو عمل ريفريش قبل ما يدخل.
3. بيظهر toast بيأكد الاختيار (مثال: "كرين C03 هيتكتب تلقائي في التاسك الجاي").
4. الزرار اللي اتدوس بياخد شكل مميز (`.active`) عشان يبان إنه هو المختار.

**الاستخدام الفعلي:** لما نفس اليوزر (W) يدخل بعد كده ويفتح مودال "New Task" (عن طريق `openNewTaskModal()`)، خانة الـ **Crane** في الفورم بتتملى تلقائيًا بالقيمة المحفوظة في `preselectedCrane` — من غير ما يحتاج يختارها تاني من الـ dropdown بتاع الفورم. يقدر يغيّرها يدويًا لو حابب قبل ما يبعت التاسك.

**ملحوظة مهمة:** الاختيار ده **مش بيتصفر لوحده** بعد أول تاسك — يعني لو الـ W شغال على نفس الكرين طول الشيفت، هيفضل نفس الرقم متكتوب تلقائي في كل تاسك جديد لحد ما يغيّره بنفسه (يكتب ID تاني في خانة الدخول ويختار كرين مختلف من السكرول بار، أو يعدّل يدويًا من جوه المودال).

## 4. موديل البيانات (Data Model)

### 4.1 التاسك (Task) — الجدول الحي

كل تاسك عبارة عن object بالشكل ده:

```js
{
  id: 'tk_<timestamp>_<random>',
  container: 'HLBU1234567',      // رقم الحاوية
  position: '22-04-82',           // البوزيشن
  crane: '3',                     // رقم الكرين (ممكن يكون متكتوب تلقائي من preselectedCrane)
  truck: 'TR15',                  // رقم التريلا
  from: 'W1201',                  // مين أنشأ التاسك
  positionOnArrival: '',          // بوزيشن الوصول — بتتحدد وقت الـ Start
  startedBy: null,                // مين من P بدأ التنفيذ
  status: 'waiting',              // waiting | inprogress | done
  createdAt: <timestamp>,
  startedAt: null,
  doneAt: null
}
```

بيتخزن مصفوفة (array) من التاسكات دي تحت مفتاح `ops-task-board-v2:tasks`.

### 4.2 الـ History Log — السجل الدائم

نفس شكل الـ Task بالظبط، بالإضافة إلى:

```js
{
  ...كل حقول الـ Task,
  deleted: false,                 // true لو التاسك اتحذف من اللوحة الحية
  deletedAt: null,                // تاريخ الحذف
  lastStatusBeforeDelete: '...'   // آخر حالة كانت للتاسك قبل الحذف
}
```

بيتخزن تحت مفتاح منفصل: `ops-task-board-v2:history`.

**الفرق الجوهري بين الاتنين:**
- **Tasks** = اللوحة الحية. لما تاسك يتحذف، بيتشال من هنا نهائيًا.
- **History** = سجل دائم. أي تاسك بيتعمل بيتسجل هنا **مرة واحدة وبيفضل موجود للأبد**، حتى لو اتحذف من الـ Tasks — بس بيتعلّم عليه `deleted: true` بدل ما يتشال.

يعني: الحذف من لوحة M بيمسح التاسك من اللوحة الحية بس، ومش بيأثر على السجل التاريخي أبدًا.

### 4.3 مفتاح الكرين المحفوظ (جديد)

`ops-task-board-v2:pending-crane` — مفتاح منفصل في localStorage بيحفظ آخر رقم كرين اختاره أي W من السكرول بار قبل الدخول. القيمة دي عبارة عن string بسيط ('1' لحد '7')، ومش مرتبطة بـ ID معين — يعني لو دخل W1201 واختار C03، وبعدين خرج ودخل W1500، هيلاقي C03 لسه متحفوظ ومقترح ليه برضه (لحد ما يغيّره).

## 5. دوال التخزين (Persistence)

| الدالة | بتعمل إيه |
|---|---|
| `loadTasks()` | بتقرأ `tasks` من localStorage |
| `persistTasks()` | بتكتب `tasks` في localStorage |
| `loadHistory()` | بتقرأ `historyLog` من localStorage |
| `persistHistory()` | بتكتب `historyLog` في localStorage |
| `addHistoryEntry(t)` | بتضيف نسخة من التاسك الجديد في `historyLog` وقت الإنشاء |
| `updateHistoryEntry(id, patch)` | بتحدّث حقول معينة في سجل تاريخي موجود (زي الحالة أو تاريخ الحذف) من غير ما تمسح السجل |

كمان فيه `window.addEventListener('storage', ...)`: ده بيخلي أي تبويب (tab) تاني مفتوح على نفس المتصفح يتحدث لحظيًا لو حصل تغيير في `tasks` أو `history` من تبويب تاني — من غير ما تحتاج ريفريش.

## 6. دورة حياة التاسك (Task Lifecycle) وربطها بالـ History

```
[W يكتب ID بيبدأ بحرف W في خانة الدخول]
     │
     └─► ييظهر سكرول بار الكرينز (C01–C07)، لو اختار واحد بيتحفظ في preselectedCrane + localStorage

[W ينشئ تاسك]
     │
     ├─► خانة Crane في المودال بتتملى تلقائي من preselectedCrane (لو موجود)
     ├─► يتضاف في tasks[]           (status: waiting)
     └─► يتضاف في historyLog[]      (deleted: false) ← addHistoryEntry()

[P يعمل Start ويكتب Position on Arrival]
     │
     ├─► يتحدّث في tasks[]           (status: inprogress, startedBy, startedAt)
     └─► يتحدّث في historyLog[]      (نفس التحديثات)   ← updateHistoryEntry()

[صاحب التاسك (W) يدوس Done]
     │
     ├─► يتحدّث في tasks[]           (status: done, doneAt)
     └─► يتحدّث في historyLog[]      (نفس التحديثات)   ← updateHistoryEntry()

[أي حد مصرح له يحذف التاسك]
     │
     ├─► يتشال نهائيًا من tasks[]
     └─► في historyLog[]: مبيتشلش — بس يتحط عليه
            deleted: true, deletedAt: Date.now()   ← updateHistoryEntry()
```

بالشكل ده، الـ History بيفضل عنده **كل تاسك اتعمل من أول ما اتفتح التول**، بغض النظر إنه اتقفل أو اتحذف.

## 7. الدوال الأساسية للأكشنز (Actions)

- **`toggleCranePicker(val)`** *(جديد)*: بتشيك لو النص المكتوب في خانة الـ ID بيبدأ بحرف W، وتظهر أو تخفي سكرول بار الكرينز على أساس كده.
- **`renderCranePicker()`** *(جديد)*: بترسم أزرار الكرينز (C01–C07) جوه السكرول بار، وبتحدد الزرار اللي متحفوظ حاليًا بشكل `.active`، وبتربط كل زرار بحدث `click` بيحفظ الاختيار.
- **`startTask(id)`**: بتتأكد إن فيه Position on Arrival متكتوب، لو فاضي بترفض وتعرض توست تنبيه. لو تمام، بتحدّث التاسك في `tasks` وفي `historyLog` مع بعض.
- **`doneTask(id)`**: بتقفل التاسك (status = done) في `tasks` و`historyLog`.
- **`deleteTask(id)`**: بتتأكد الأول من الصلاحية (W يقدر يحذف تاسكاته بس)، وبتاخد تأكيد (`confirm`) قبل الحذف، وبعدين تمسح من `tasks` وتعلّم عليه `deleted:true` في `historyLog`.
- **`openNewTaskModal()`**: بتفتح مودال فيه فورم (Container / Position / Crane / Truck)، وبتملي خانة الـ Crane تلقائيًا من `preselectedCrane` لو موجودة، وبتتأكد إن كل الحقول متملية قبل الإرسال، وبتنشئ الـ object بتاع التاسك الجديد.

## 8. الواجهات حسب الدور (Views)

### 8.1 `renderWView()` — واجهة W
- بتعرض بس التاسكات اللي `from === myId` (يعني اللي هو نفسه أنشأها).
- زرار "New Task" لفتح المودال (اللي بيبقى فيه الكرين متكتوب تلقائي لو المستخدم اختار واحد قبل الدخول).
- كل كارت فيه زرار "Done" (لو In Progress) وزرار "حذف".

### 8.2 `renderPView()` — واجهة P
- قسم "تاسكات جديدة (Waiting)": بيعرض كل التاسكات اللي حالتها Waiting مع حقل Position on Arrival وزرار Start.
- قسم "التاسكات اللي بدأتها إنت (In Progress)": بيعرض بس اللي هو بدأها (`startedBy === myId`).

### 8.3 `renderMVView()` — واجهة M و V
دي أهم دالة اتعدلت عشان تحقق طلب الـ History والـ Export:

- **لو الدور M**: بتاخد البيانات من `historyLog` (السجل الدائم) مش من `tasks`، يعني بتعرض **كل** الحاويات اللي اتعمل عليها تاسك من أول ما اتفتح التول، بما فيها اللي اتحذفت (وبتتعلّم بعلامة "محذوفة" وتاريخ الحذف). وبيظهر زرار **⬇ Export Excel** فوق.
- **لو الدور V**: بتاخد البيانات من `tasks` بس (اللوحة الحية الحالية)، من غير حذف ولا Export — مشاهدة بس زي دوره بالظبط.
- الإحصائيات (`statsHtml`) بتتحسب على التاسكات الغير محذوفة بس، عشان الأرقام تعكس الوضع الحالي مش السجل الكامل.

## 9. تصدير البيانات (Export) — وظيفة `exportHistoryCsv()`

متاحة لدور **M** بس. بتعمل الآتي:

1. بتاخد كل السجلات من `historyLog` مرتبة من الأقدم للأحدث.
2. بتبني ملف **CSV** فيه الأعمدة دي بالظبط (الهيدر بقى **بالإنجليزي بالكامل** — تحديث جديد):

   | العمود (Header) | المصدر |
   |---|---|
   | `Container Number` | `container` |
   | `Position` | `position` |
   | `Position on Arrival` | `positionOnArrival` |
   | `Status` | `status` (Waiting / In Progress / Done) |
   | `Deleted?` | `Yes` / `No` حسب `deleted` |
   | `Created By` | `from` |
   | `Started By` | `startedBy` |
   | `Created At` | `createdAt` (متنسق بالتاريخ والوقت) |

3. بتضيف BOM (`\uFEFF`) في أول الملف عشان اللغة العربية (لو موجودة في محتوى الأعمدة زي رقم البوزيشن) تتفتح صح في Excel من غير حروف مشوهة.
4. بتعمل `Blob` وتنزل الملف تلقائيًا باسم:
   `containers-history-<تاريخ اليوم>.csv`

ملاحظة: الملف CSV بيتفتح مباشرة في Excel/Google Sheets زي أي شيت عادي، ومفيهوش أي احتياج لبرنامج إضافي. **الهيدر وقيم عمود "Deleted?" بقيا بالكامل بالإنجليزي (Yes/No) عشان يبقوا متسقين مع باقي الأعمدة.**

## 10. حدود مهمة لازم تعرفها

- **التخزين محلي فقط (per-browser/per-device)**: مفيش سيرفر ولا قاعدة بيانات مركزية. لو حد فتح نفس اللينك من جهاز تاني، هيلاقي الـ Tasks والـ History **فاضيين** عنده، لأنهم مش متزامنين بين الأجهزة.
- الحاجة الوحيدة اللي بتتزامن هي بين تبويبات (tabs) مفتوحة على **نفس المتصفح ونفس الجهاز**، عن طريق `storage` event.
- لو حد مسح بيانات المتصفح (Clear browsing data) أو استخدم وضع تصفح خفي (Incognito)، هيفقد كل الـ Tasks والـ History المحفوظة، **وكمان هيفقد آخر كرين محفوظ (`pending-crane`)**.
- الكرين المحفوظ (`preselectedCrane`) **مش مربوط بـ ID معين** — هو مجرد "آخر اختيار" على مستوى المتصفح، فلو أكتر من W بيستخدموا نفس الجهاز، هيشتركوا في نفس القيمة المقترحة (يقدروا يغيّروها بسهولة من السكرول بار قبل ما يبعتوا التاسك).
- لو المطلوب مستقبلًا إن كل المستخدمين (على أجهزة مختلفة) يشوفوا نفس البيانات لحظيًا، محتاجين ننقل التخزين لقاعدة بيانات حقيقية بدل localStorage (ده تغيير أكبر في البنية).

```
loadHistory / persistHistory

```
